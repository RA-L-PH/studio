# PulseQueue Firebase Security Rules (MVP Version)

## Firestore Rules
# Location: firestore.rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Global permissive rules for MVP development
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

## Realtime Database Rules
# Apply these in the 'Rules' tab of the Realtime Database section in the Firebase Console.
{
  "rules": {
    ".read": "true",
    ".write": "true"
  }
}

## Implementation Notes:
1. These rules are configured for an MVP to allow public access for rapid testing.
2. For production, replace '.read: true' and '.write: true' with auth-based checks (e.g., 'auth != null').
3. Ensure you have initialized your Realtime Database in the Firebase Console.
