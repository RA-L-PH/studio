
# PulseQueue | Full Stack Real-Time Clinic Flow

PulseQueue is a high-performance, real-time medical queue management system designed for modern clinics. This project demonstrates advanced **Full Stack Development** capabilities, focusing on **Backend Engineering** and **Real-Time Database Management**.

## 🚀 Architectural Context

While implemented using **Next.js** and **Firebase Realtime Database**, the architectural principles of PulseQueue draw from extensive experience in:

- **Backend Development**: Expertise in **Django** and **Express.js** for building scalable, secure server environments.
- **Database Management**: Proficiency in **MongoDB** and **PostgreSQL**, moving beyond basic CRUD to complex atomic operations and distributed state sync.
- **Full Stack Integration**: Seamlessly connecting low-latency frontend interfaces (React 19 + Tailwind) with robust backend logic.

## 🛠 Key Features

- **Ultra-Low Latency Sync**: Powered by Firebase RTDB for sub-second updates between Reception, Clinician, and Patient displays.
- **"Neu-Glass" UI**: A custom design system blending Neumorphism for depth and Glassmorphism for clarity.
- **Predictive Metrics**: Real-time throughput calculations to estimate patient wait times dynamically.

## 💻 Tech Stack Highlights

- **Frontend**: Next.js 15 (App Router), React 19, ShadCN UI, Tailwind CSS.
- **Backend**: Firebase RTDB, Next.js Server Actions.
- **Database**: Realtime Database for live state; previously architected for **MongoDB** / **Express.js** compatibility.
